# claude-cognitive-os

**Автор:** rakhman
**Язык:** русский

Когнитивная операционная система для Claude. Превращает любой проект в структурированное рабочее пространство с автоматическим извлечением знаний, визуальным графом связей и самонастраивающимися порогами.

## Что делает плагин

- **Структурированная память проектов** — 15 карточек-шаблонов покрывают все аспекты проекта: от текущего состояния до обнаруженных паттернов и ошибок
- **Автоматическое извлечение паттернов** — детектор фиксирует повторяющиеся паттерны из 3+ случаев и выносит их в `02_patterns.md`
- **Визуальный граф связей** — интерактивный HTML-граф на vis-network со всеми сущностями, картами знаний, правилами и проектами
- **Двусторонние обратные ссылки** — backlinks обновляются автоматически после каждой записи
- **Автокалибровка порогов** — пороги (systemic_threshold, min_occurrences, ...) подстраиваются по реальным данным
- **Self-reflection workflow** — после каждой комплексной задачи: оценка 0-100%, журнал решений, уроки
- **Верификация П1-П14** — встроенная система из 32 механизмов и 19 пресетов
- **Интеграция с Obsidian** — нативные `[[wikilinks]]`, frontmatter YAML, совместимость с Graph View

## Установка

Установить `.plugin` файл в Cowork или Claude Code: одно нажатие в rich preview → плагин активируется.

## Обновление

При выходе новой версии плагина — установить новый `.plugin` файл поверх предыдущего. Cowork/Claude Code заменит исходники плагина в `/var/folders/.../claude-hostloop-plugins/<id>/`, воркспейсы (`cognitive_os/`) не трогаются. Актуальная версия — в `.claude-plugin/plugin.json`, разбор изменений — в [CHANGELOG.md](./CHANGELOG.md).

Миграция между мажорными версиями (например, v1 → v2): запустить `migrate` в action-скиллах. Для минорных и патчей миграция не требуется.

## Использование

Скиллы вызываются автоматически по триггерам в запросах. Также доступны как явные действия:

| Action-скилл | Что делает |
|---|---|
| `init` | Инициализация cognitive-os в текущей папке (создаёт `CLAUDE.md` + пустые шаблоны) |
| `migrate` | Миграция существующего проекта из произвольной структуры в формат cognitive-os |
| `status` | Отчёт о состоянии текущего проекта: сколько карточек, паттернов, ошибок, размер графа |
| `graph` | Пересборка визуального графа `_generated/graph.html` (vis-network) |
| `backlinks` | Пересборка файла обратных ссылок `_generated/backlinks.md` для AI-чтения |
| `reflect` | Запуск self-reflection после комплексной задачи |
| `calibrate` | Автокалибровка порогов на основе накопленных данных |
| `archive` | Архивация старых сессий в `archive/YYYY-MM/` |
| `patterns` | Обзор всех обнаруженных паттернов с ранжированием по частоте и уверенности |
| `audit` | Аудит противоречий и скрытых допущений во всём vault'е |

## Три сценария использования

### 1. Только плагин (без Obsidian)

Работает самостоятельно. Все карточки в `.md`, можно читать любым редактором. Без визуального графа в реальном времени, но всё остальное — полностью.

### 2. Только Obsidian (без плагина)

Стандартная Obsidian-работа с заметками. Claude читает файлы как есть, без когнитивного слоя плагина.

### 3. Комбо (рекомендуется)

Одна папка на диске → Obsidian как интерфейс для вас, плагин как мозжечок для Claude. Оба смотрят на одну и ту же файловую систему. См. `skills/cognitive-os-core/references/integration_with_verification.md` и `docs/presets_reference.md`.

## Рекомендуемая структура vault'а

```
~/Documents/                              ← Obsidian Vault
├── Claude/
│   ├── Plugins/                          ← exclude в Obsidian
│   │   └── claude-cognitive-os/          ← исходники плагина
│   └── Projects/
│       ├── Jarvis/                       ← один проект = одна папка
│       │   ├── 00_index.md
│       │   ├── 01_current_state.md
│       │   ├── ...
│       │   └── _generated/               ← exclude в Obsidian
│       │       ├── graph.html
│       │       └── backlinks.md
│       └── <другие проекты>/
```

## Настройки Obsidian для комбо-сценария

Перед первым использованием:

1. **Options → Файлы и ссылки → Место для новых заметок** = "Та же папка, что и текущий файл"
2. **Options → Файлы и ссылки → Всегда обновлять внутренние ссылки** = ВКЛ (критично!)
3. **Options → Файлы и ссылки → Исключённые файлы** — добавить:
   - `Claude/Plugins`
   - `**/_generated`
   - `**/.claude-plugin`
4. **Options → Файлы и ссылки → Использовать [[Wiki-ссылки]]** = ВКЛ (обычно уже включено)

См. `examples/starter_workspace/README.md` и `docs/thresholds_reference.md`.

## Архитектура

Плагин состоит из:

- **3 knowledge-скилла** — ядро знаний, граф, калибровка
- **10 action-скиллов** — init, migrate, status, graph, backlinks, reflect, calibrate, archive, patterns, audit
- **3 субагента** — cog-archivist (архивация), cog-detector (детекция паттернов), cog-verifier (верификация)
- **1 хуков-файл** — post-write hook с debounce для автопересборки графа/backlinks
- **10 Python-скриптов** — рендеринг графа/backlinks, детектор сигналов, калибровка, пагинация, sync-check, аудит противоречий
- **15 шаблонов карточек** — от `00_index.md` до `14_audit_log.md`
- **5 документов** — `docs/architecture.md`, `docs/migration_guide.md`, `docs/quick_start.md`, `docs/thresholds_reference.md`, `docs/presets_reference.md` (+ `docs/verification/` — отчёты верификации)
- **1 starter workspace** — пример инициализации нового проекта с `.obsidian/` конфигом

Детали в `docs/architecture.md` и `docs/quick_start.md`.
