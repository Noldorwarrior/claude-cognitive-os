# reflect — Самоаудит по 8 осям

## Назначение

После завершения комплексной задачи провести самоаудит Claude по
8 осям (Корректность, Полнота, Формат, Источники, Логика, Аудитория,
Калибровка, Уроки), зафиксировать результат в [[13_self_reflection]],
извлечь уроки / ошибки / паттерны.

## Когда запускается

- Автоматически после завершения задачи, отмеченной
  `requires_reflection: true` (настраивается в SKILL.md задачи).
- Явная команда «самоанализ» / «оцени свою работу» / «reflect».
- Задача в [[productivity:task-management]] отмечена `[x]` — срабатывает
  post-edit хук `trigger_reflect_on_tasks.py` на `TASKS.md`
  (зарегистрирован в `hooks/hooks.json`, дебаунс 5 сек,
  реагирует только на прирост количества `[x]`).
- Monthly aggregation: раз в месяц — агрегированный отчёт.

## Когда НЕ запускается

- Простые/короткие задачи (1 вопрос — 1 ответ, без файлов).
- Задача ещё не завершена (нет итогового результата).
- Пользователь явно запросил «без самоаудита».

## Workflow

### 0. Pre-flight

Перед запуском самоаудита проверить два **независимых** условия. Не
объединять их провалы в общий диалог с `init` первым пунктом — это
историческая ловушка: `init` в неправильной cwd создавал дубль-пустышку
вместо указания на существующий воркспейс.

**0.1. Воркспейс доступен?**

Проверка: `${CLAUDE_WORKSPACE:-.}` существует и содержит `00_index.md`.

- Если **доступен** — продолжить.
- Если **не найден** — вывести пользователю диалог с **одним** первым
  рекомендуемым действием:

  ```
  Воркспейс cognitive_os не найден.

  Рекомендуемое действие — указать существующий воркспейс через
  переменную окружения:

    export CLAUDE_WORKSPACE=/path/to/cognitive_os

  См. docs/quick_start.md для подробностей и примера для ~/.zshrc.

  Создание нового воркспейса через [[init]] — отдельная осознанная
  команда. Запускайте её только если действительно хотите развернуть
  новую структуру с нуля. В этом fallback-меню `init` намеренно
  исключён, чтобы не создать дубль-пустышку в неправильной папке.
  ```

  Прервать reflect. Пользователь сам решает: выставить переменную или
  запустить `init` отдельной командой.

**0.2. Есть задача для самоаудита?**

Проверка: в контексте сессии есть завершённая задача ИЛИ в
`TASKS.md` есть недавний `[x]` ИЛИ передан явный флаг
`--task=<id>` / `--month=<YYYY-MM>` / `--domain=<X>`.

- Если **задача есть** — продолжить.
- Если **задачи нет** — вывести подсказку по флагам:

  ```
  Не удалось определить задачу для самоаудита.

  Укажите явно один из режимов:
    reflect --task=<id>            — конкретная задача из TASKS.md
    reflect --month=<YYYY-MM>      — агрегированный отчёт за месяц
    reflect --domain=<domain>      — агрегированный отчёт по домену

  Либо завершите задачу в текущей сессии (поставьте `[x]` в TASKS.md
  или явно обозначьте итог) — тогда reflect подхватит её автоматически
  через хук `trigger_reflect_on_tasks.py`.
  ```

  Прервать reflect. Решение — за пользователем.

**Важно:** 0.1 и 0.2 — независимые проверки. Для `reflect
--month=<YYYY-MM>` нужен воркспейс (чтобы читать `13_self_reflection`),
для ad-hoc запуска без параметров — только контекст задачи. Поэтому
проверять в порядке 0.1 → 0.2 и выходить после первого провала, не
смешивая диалоги.

### 1. Выбор режима

**Приоритет:**

1. Если установлен [[govdoc-analytics:self-reflection]] → **делегировать**
   туда. Полноценный отчёт 8 осей.
2. Если НЕ установлен → **fallback к internal_8axis_scoring** (упрощённая
   эвристика внутри этого скилла).

Определение через Skill tool: попробовать вызвать `govdoc-analytics:self-reflection`,
при ошибке — fallback.

### 2. Сбор данных о задаче

- **Задача:** из TASKS.md или контекста.
- **Проект:** если есть — [[proj-NNN]].
- **Длительность:** оценка по timestamp сессии.
- **Инструменты использованные:** из истории вызовов.
- **Режим верификации:** М1-М5 / П1-П14 / не применялась.

### 3. Режим A — external (govdoc-analytics:self-reflection)

Делегировать в скилл, передав:
- Текст задачи + результат.
- Контекст проекта.
- Применённые инструменты и верификации.

Получить структурированный отчёт. Скопировать в [[13_self_reflection]] как
запись `sr-NNN`.

### 4. Режим B — fallback (internal_8axis_scoring)

Упрощённая эвристика по 8 осям:

| Ось | Эвристика |
|---|---|
| Корректность | Сверка цифр/дат/имён с исходниками (М1) |
| Полнота | Проверка по чек-листу пунктов запроса (М2) |
| Формат | Сверка с [[01_user_profile]] (правила #6, #7) |
| Источники | Есть ли явные ссылки? |
| Логика | Поиск противоречий (М7) |
| Аудитория | Моделирование получателя (М27) |
| Калибровка | Гэп заявл./факт. ≤ 10%? |
| Уроки | Зафиксирован ли хотя бы 1 lesson / ec / insight? |

Оценка каждой оси: 0-100%. Средний скор — средняя.

### 5. Запись в 13_self_reflection

Новая запись `sr-NNN` по шаблону из [[13_self_reflection]]:

- Дата, проект, задача, длительность, инструменты.
- Режим: `external` / `internal_8axis_scoring`.
- Скоринг по 8 осям (таблица).
- Средний скор.
- Журнал решений (основные выборы).
- Уверенность (заявленная / фактическая / подтверждённая).
- Извлечённые уроки → ссылки на `lesson-NNN` в [[06_lessons_learned]].
- Обнаруженные ошибки → ссылки на `ec-NNN` в [[10_error_corrections]].
- Применённая верификация.
- Дрейф смысла (если были итерации — М26).

### 6. Запись в 11_confidence_scoring

Добавить задачу в журнал домена:
- Дата, задача, заявленная уверенность, фактическая, расхождение, ошибка?
- Обновить средние метрики домена.
- Если средний гэп > +10% — поднять флаг для [[calibrate]].

### 7. Извлечение артефактов

**Lessons:** если в ходе задачи Claude извлёк вывод → записать в
[[06_lessons_learned]] как `lesson-NNN`. Прикрепить ссылку на `sr-NNN`.

**Errors:** если Claude обнаружил свою ошибку → записать в
[[10_error_corrections]] как `ec-NNN`. Обязательная верификация П5.

**Patterns (кандидаты):** если Claude увидел повторяющееся наблюдение →
записать в `06_lessons_learned` с пометкой «кандидат в паттерн».
Если счётчик повторений ≥ `thresholds.pattern` → триггерит [[patterns]].

**Meta-decisions (кандидаты):** если накопилось ≥3 lessons на одну тему →
предложение в [[04_meta_decisions]] через `extract_principles.py`.

### 8. Обновление 00_index

Обновить счётчики в `00_index.md`:
- `active_count` в тех карточках, куда добавлены записи.
- `last_event` в событиях.

### 9. Monthly aggregation

Если команда `reflect --month=<month>` — построить сводный отчёт:

```markdown
# Самоаудиты за апрель 2026

Всего задач: 18
Средний скор по 8 осям:
| Ось | Средний |
|---|---|
| Корректность | 93% |
| Полнота | 89% |
| Формат | 96% |
| Источники | 85% |
| Логика | 92% |
| Аудитория | 88% |
| Калибровка | 81% |
| Уроки | 95% |

**Слабейшая ось:** Калибровка (81%) — систематически переоцениваю
уверенность. Рекомендация: снизить заявленную уверенность на 10% в
доменах `govsector-documents` и `finance-analytics`.

**Топ-5 уроков:** [[lesson-018]], [[lesson-021]], ...
**Топ-5 ошибок:** [[ec-012]], [[ec-013]], ...
```

Сохранить в `docs/monthly_reflections/{{YYYY-MM}}.md`.

## Режимы

### `reflect` (default)

Самоаудит текущей завершённой задачи.

### `reflect --task=<id>`

Самоаудит конкретной задачи (из TASKS.md).

### `reflect --month=<YYYY-MM>`

Monthly aggregation.

### `reflect --domain=<domain>`

Агрегированный самоаудит по домену.

### `reflect --short`

Упрощённая форма (без 8 осей, только выводы).

## Команды

| Команда | Действие |
|---|---|
| `самоанализ` | Текущая задача |

## Верификация

**П1 «Аналитик»** для самого reflect: проверка, что все 8 осей
заполнены, связи к lesson/ec/sr проставлены, калибровка зафиксирована.

## Частые ошибки

1. **Пропущены связи** (sr-NNN без ссылок на lessons) → снижает ценность.
   Защита: шаг 7 обязателен.
2. **Overconfident в калибровке** (заявляю 95%, реально 75%) → систематическое
   смещение. Защита: сверка с прошлыми домен-задачами, auto-flag в calibrate.
3. **Дубль-самоаудит** (за одну задачу два sr-NNN) → путаница. Защита:
   проверка по `task_id` перед записью.
