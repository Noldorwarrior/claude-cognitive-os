# calibrate — Калибровка уверенности и порогов

## Назначение

Превратить накопленные данные из [[13_self_reflection]] и
[[11_confidence_scoring]] в конкретные рекомендации:

1. **Калибровка уверенности** — снизить/повысить заявленную уверенность
   в доменах, где систематический гэп.
2. **Калибровка порогов** — пересмотреть `thresholds.*` в воркспейсе,
   если они не отражают реальную частоту / качество.
3. **Активация обязательной П5** — в доменах с низкой фактической
   успешностью.

Результат — предложения в [[04_meta_decisions]], которые пользователь
одобряет или отклоняет.

## Когда запускается

- Явная команда «откалибруй» / «калибровка» / «calibrate».
- Автоматически раз в `revision_interval_days` (default 30).
- После [[reflect]] если обнаружен сильный гэп (> 20% на задачу).
- После 10+ новых записей в [[13_self_reflection]].

## Когда НЕ запускается

- Недостаточно данных (< `thresholds.confidence_min_tasks` задач в любом
  домене).
- Калибровка уже была < 7 дней назад (кроме принудительного режима).

## Workflow

### 1. Сбор данных

Читает:
- `11_confidence_scoring.md` — все домены и их журналы задач.
- `13_self_reflection.md` — все sr-NNN с калибровкой.
- `02_patterns.md` — активные паттерны и их использование.
- `10_error_corrections.md` — частота ошибок по доменам.

### 2. Расчёт калибровочных метрик

Для каждого домена с ≥ `confidence_min_tasks` задач:

| Метрика | Расчёт | Назначение |
|---|---|---|
| `mean_declared` | mean(declared) | Средняя заявленная уверенность |
| `mean_actual` | mean(actual) | Средняя фактическая успешность |
| `gap` | mean(declared − actual) | Систематическое смещение (знаковое) |
| `std_gap` | std(declared − actual) | Устойчивость калибровки |
| `MAE` | mean(\|declared − actual\|) | Средняя абсолютная ошибка |
| `overconfidence_rate` | fraction(declared > actual) | Доля переоценки |
| `underconfidence_rate` | fraction(declared < actual) | Доля недооценки |
| `critical_errors` | count(ec-*.severity ∈ {high, critical}) | Активация П5 |
| `recurring_errors` | count(ec-*.status = recurring) | Систематические ошибки |

`gap/std_gap` нужны для детекции направленного смещения (шаг 3);
`MAE/overconfidence_rate/underconfidence_rate` — согласованы с
[[cognitive-os-calibration]] и используются скриптом
`calibrate_thresholds.py`. Пороги интерпретации MAE:

- `MAE < 0.10` — отличная калибровка, без корректировки.
- `MAE 0.10-0.20` — приемлемо, мягкая корректировка.
- `MAE > 0.20` — систематическое смещение, жёсткая корректировка.
- `mean_actual < 0.7` — критично, активировать обязательную П5 (шаг 5).

### 3. Детекция overconfidence / underconfidence

- **Overconfident:** гэп > +10%, |std| < 15% (стабильный гэп).
- **Underconfident:** гэп < −10%, |std| < 15%.
- **Неустойчивая калибровка:** |std| > 20% — данные шумные, требуется
  больше задач.

### 4. Пересмотр порогов

Для каждого порога в `00_index.thresholds`:

| Порог | Сигнал к пересмотру |
|---|---|
| `pattern: 3` | Если паттернов > `max_active_patterns` (50) — повысить до 4 |
| `pattern_window_days: 30` | Если паттерны устаревают быстрее — снизить |
| `max_active_patterns: 50` | Если постоянное упирание в лимит — повысить |
| `max_active_lessons: 30` | Аналогично |
| `cluster_min_projects: 3` | Если кластеров нет при 10+ проектах — снизить до 2 |
| `systemic_error: 3` | Если ec-recurring много — снизить до 2 |
| `confidence_min_tasks: 5` | Если домены не активируются — снизить |

### 5. Активация обязательной П5

Если в домене:
- `mean(actual) < 70%`
- ИЛИ `count(critical_errors) ≥ 2`
- ИЛИ `count(recurring_errors) ≥ 1`

→ Рекомендация: для этого домена все новые задачи автоматически идут с
верификацией **П5 «Максимум»** (32 механизма).

Снимается, когда `mean(actual) ≥ 80%` в течение 3 месяцев.

### 6. Генерация рекомендаций

Формат результата — предложения, требующие одобрения пользователя:

```markdown
# Калибровка — {{DATE}}

## Overconfidence

- **`govsector-documents`** — заявляю 87%, факт 71%, гэп +16% (высокий).
  - Рекомендация: заявлять 75% в этом домене.
  - Обоснование: 18 задач, стабильный гэп (std 9%).
  - Статус: ⚠️ ожидает одобрения.

## Underconfidence

- **`presentations`** — заявляю 70%, факт 85%, гэп −15%.
  - Рекомендация: заявлять 82%.
  - Обоснование: 9 задач, стабильный (std 7%).

## Пересмотр порогов

- **`cluster_min_projects`** — сейчас 3, кластеров 0 при 8 проектах.
  - Рекомендация: снизить до 2.

## Обязательная П5

- **`finance-analytics`** — mean(actual) = 62%, 2 critical ошибки.
  - Рекомендация: активировать обязательную П5 для домена.

## Снятие обязательной П5

- **`govsector-speech`** — в P5-режиме 4 месяца, последние 3 месяца
  mean(actual) = 88%.
  - Рекомендация: снять обязательную П5.
```

### 7. Диалог с пользователем

Через `AskUserQuestion`:

«Принять все рекомендации / выбрать по одной / отклонить все?»

Каждая принятая → запись в [[04_meta_decisions]] как `md-NNN` с типом
`threshold` или `methodology`.

### 8. Применение изменений

- Изменить `thresholds.*` в `00_index.md` (и соответствующих карточках).
- Запустить `sync_check.py` для распространения по всем файлам.
- Зафиксировать в `docs/calibration_log.md`.

### 9. Уведомление

```
Калибровка завершена.
Принято предложений: 5 из 7.
Изменено порогов: 2.
Активирована обязательная П5 для: finance-analytics.
Снята П5 для: govsector-speech.
Следующая калибровка: 2026-05-20.
```

## Режимы

### `calibrate` (default)

Полная калибровка всех доменов и порогов.

### `calibrate --dry-run`

Показать рекомендации без применения.

### `calibrate --domain=<domain>`

Только один домен.

### `calibrate --thresholds-only`

Только пересмотр порогов, без уверенности.

### `calibrate --force`

Запустить даже если < 7 дней с последней.

| Команда | Действие |
|---|---|

## Верификация

**П3 «Бухгалтер»** — для калибровки:
- М20 двойной расчёт гэпа.
- М3 сверка сумм (средние значения).
- М4 проверка границ (0-100%).
- М22 согласованность файлов после применения.

## Частые ошибки

1. **Мало данных** (< 5 задач в домене) → ложные рекомендации.
   Защита: минимальный порог + предупреждение.
2. **Применение без одобрения** → молчаливое изменение порогов.
   Защита: обязательный AskUserQuestion в шаге 7.
3. **Рассинхрон после изменения** (один файл обновлён, другой нет).
   Защита: `sync_check.py` в шаге 8.
