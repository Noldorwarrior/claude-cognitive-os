# patterns — Анализ и извлечение паттернов

## Назначение

Обнаруживать повторяющиеся наблюдения в работе Claude и формализовывать
их как паттерны → механизмы защиты → мета-принципы:

```
наблюдение (в lessons)
  ↓ 3+ раз в окне 30 дней
pat-NNN (в 02_patterns)
  ↓ если нужна защита от ошибок
wm-NNN (в 09_working_mechanisms)
  ↓ 3+ паттернов на одну тему
md-NNN (в 04_meta_decisions как принцип)
```

## Когда запускается

- Явная команда («покажи паттерны», «извлеки паттерны», «что я делаю
  систематически»).
- Автоматически после [[reflect]], если накопились новые наблюдения.
- Автоматически при срабатывании порога (`thresholds.pattern = 3`).
- Post-write hook на `06_lessons_learned.md` (с debounce).

## Когда НЕ запускается

- Мало данных (< 10 записей в `06_lessons_learned`).
- Запущен другой процесс модификации карточек.

## Workflow

### 1. Сбор наблюдений-кандидатов

Читаем [[06_lessons_learned]] и собираем:
- Все lessons, возраст которых ≤ `pattern_window_days` (default 30).
- Группируем по семантической близости (триггер, контекст, тема).
- Считаем повторения.

### 2. Детекция кандидатов в паттерны

Наблюдение становится **кандидатом** при:
- `repetitions ≥ thresholds.pattern` (default 3).
- В разных контекстах (не один проект).
- Без уже существующего pat-NNN, покрывающего это.

Пример:

```
Наблюдение: «Пользователь всегда добавляет пре-мортем после SWOT»
Повторений: 4
Контексты: proj-001 (концепция), proj-005 (стратегия), proj-008 (программа), proj-012 (форум)
→ кандидат pat-NNN
```

### 3. Проверка на дубликаты / подмножества

- Fuzzy-сверка с существующими pat-NNN.
- Если похож на существующий → предложить **расширить существующий** или
  **создать новый с cross-link**.

### 4. Предложение пользователю

Через `AskUserQuestion`:

```
Обнаружен кандидат в паттерн:
«Пре-мортем после SWOT в концепциях»

Повторений: 4 (proj-001, proj-005, proj-008, proj-012).
Категория: methodology.

Варианты:
A. Зафиксировать как pat-NNN (добавить в 02_patterns).
B. Расширить существующий pat-012 (похожий).
C. Отложить (вернуться позже).
D. Отклонить (не считать паттерном).
```

### 5. Создание записи pat-NNN

Если одобрено:
- Новая запись в [[02_patterns]] по шаблону.
- Категория, триггер, повторения, контексты, связанные lessons.
- Верификация **П13 «Аудитор»** перед активацией.

### 6. Предложение механизма защиты (wm-NNN)

Если паттерн предохраняет от ошибок или ухудшения качества:

```
pat-NNN зафиксирован.

Создать механизм защиты wm-NNN?
(Механизм — правило, которое Claude применяет автоматически при
обнаружении триггера.)

Пример механизма для вашего pat-NNN:
«Перед завершением концепции всегда предлагать pre-mortem, даже если
не просили явно».

Создать? [Да / Позже / Нет]
```

При Да → запись в [[09_working_mechanisms]] по шаблону.

### 7. Извлечение мета-принципов

Периодически (раз в месяц или при срабатывании):

Если ≥ `thresholds.extract_principle_min` (default 3) паттернов на одну
тему → кандидат в `md-NNN` (meta-decision / принцип).

Через `extract_principles.py`:

```
Найдено: 3 паттерна на тему «концепции и стратегии»:
- pat-007 (pre-mortem после SWOT)
- pat-011 (backcasting первым)
- pat-015 (3 сценария всегда)

Вывод-кандидат (md-NNN): «Концепции и стратегии всегда проходят через
последовательность: SWOT → backcasting → 3 сценария → pre-mortem».

Зафиксировать как принцип? [Да / Нет / Изменить формулировку]
```

При Да → запись в [[04_meta_decisions]] как `md-NNN` типа `principle`.

### 8. Архивация устаревших паттернов

Для записей в `02_patterns`:
- `last_seen > archive_pattern_idle_months` (6) → кандидат в архив.
- Делегирование в [[archive]].

### 9. Уведомление

```
Анализ паттернов завершён.
Создано pat-*: 2 (pat-008, pat-009).
Создано wm-*: 1 (wm-012).
Кандидатов в md: 1 (ждёт одобрения).
Архивировано: 1 (pat-003).
```

## Режимы

### `patterns` (default)

Полный цикл: обнаружение + предложения + создание.

### `patterns --detect-only`

Только обнаружить кандидатов, ничего не создавать.

### `patterns --category=<category>`

Только один из типов: behavior, format, methodology, tool_usage, escalation.

### `patterns --extract-principles`

Только шаг 7 — извлечение мета-принципов.

### `patterns --review`

Провести ревью всех активных pat-NNN: какие устарели, какие нужно обновить.

### `patterns --dry-run`

Полный цикл обнаружения + показ предложений (кандидаты в pat/wm/md, план
архивации устаревших) **без записи в карточки и без AskUserQuestion**.
Используется для проверки «что произойдёт», согласования плана перед
настоящим запуском и в автоматических отчётах [[cog-detector]].

## Частые ошибки

1. **Преждевременный паттерн** (2 повторения, не 3) → ложные паттерны.
   Защита: строгая проверка порога.
2. **Дубликат** (одно и то же разными словами). Защита: fuzzy-match в шаге 3.
3. **Паттерн без механизма** — польза теряется. Защита: шаг 6 обязателен,
   но пользователь может отложить.
4. **Overfitting на один проект** — паттерн, проявившийся только в одном
   проекте, не паттерн. Защита: требование разных контекстов в шаге 2.
