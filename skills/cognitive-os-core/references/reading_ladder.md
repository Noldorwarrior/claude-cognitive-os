# Reading Ladder — уровни чтения карт

Claude читает карты vault'а в строгом порядке, останавливаясь как
только получен достаточный контекст для ответа. Это экономит токены
и сохраняет фокус.

## Уровни (v1.3)

| Уровень | Что читается | Размер | Останавливаемся, когда |
|---|---|---|---|
| -1 | Только `00_index.md::yaml` + `_generated/graph.mermaid` | ≈500 байт | Обзорный запрос о состоянии vault'а |
| 0 | YAML-frontmatter `00_index.md` | ≈100 байт | Простой вопрос без проектного контекста |
| 1 | Полный `00_index.md` + `01_user_profile.md` | ≈3 KB | Запрос про стиль / общие предпочтения |
| 2 | + `02_patterns.md` + `04_meta_decisions.md` | ≈10 KB | Выбор подхода / методологии |
| 3 | + `08_knowledge_maps.md` + `09_working_mechanisms.md` | ≈20 KB | Содержательная задача в знакомом домене |
| 4 | + все остальные карты по релевантности | до ≈50 KB | Глубокие аудиты или мета-задачи |

**Никогда не читать всё сразу.** Это противоречит цели экономии
токенов.

## Уровень -1 (новое в v1.3): обзорный

**Когда:** пользователь спрашивает «что у нас есть», «покажи состояние»,
«какие проекты в vault'е», «что нового за неделю», запускает action-скилл
`status`.

**Что читать:**
1. YAML-frontmatter `00_index.md` — метаданные: дата последнего
   обновления, общее количество карт, активных паттернов, записей в
   реестре проектов.
2. `_generated/graph.mermaid` — компактное текстовое представление
   связей (≈200-400 байт) для выдачи пользователю как преview.

**Когда останавливаться:** если пользователю нужна только сводка —
ответить по этим двум источникам. Если нужны детали — перейти на
уровень 0 или выше.

**Пример:**
> Пользователь: «Сколько у нас активных паттернов?»
> Claude читает уровень -1 → `00_index.md::yaml.active_patterns: 27` →
> Ответ: «27 активных паттернов, последняя фиксация 18.04.2026».

## Уровень 0: минимальный

**Когда:** простой вопрос, не связанный с проектами, стилем или
методологией.

**Что читать:** YAML-frontmatter `00_index.md` (≈100 байт) —
достаточно чтобы понять, инициализирован ли плагин и когда обновлялся.

**Пример:**
> Пользователь: «Как тебя зовут?»
> Claude: уровень 0 — index есть, cognitive_os активен, но конкретно
> этот запрос не требует чтения карт. Ответ: «Claude, Claude Opus 4.7».

## Уровень 1: профиль

**Когда:** запрос касается стиля работы, предпочтений, формата.

**Что читать:**
1. Полный `00_index.md` — реестр карт и перекрёстные ссылки.
2. `01_user_profile.md` — стиль, предпочтения, 10 правил
   пользователя в нормализованном виде.

**Пример:**
> Пользователь: «Напиши отчёт коротко».
> Claude: уровень 1 → `01_user_profile.md::style: concise_prose,
> lists_avoided` → Ответ без лишних списков, прозой.

## Уровень 2: подход / методология

**Когда:** нужно выбрать подход к задаче (SWOT, декомпозиция, аналитика
данных, структура презентации).

**Что читать:** + `02_patterns.md` (какие подходы пользователь
предпочитает), `04_meta_decisions.md` (какие архитектурные решения уже
принимались).

**Пример:**
> Пользователь: «Составь стратегический план».
> Claude: уровень 2 → `02_patterns.md::methodology::pat-007: SWOT first`
> + `04_meta_decisions.md::md-003: horizon 3 года, 3 сценария» →
> Применяет SWOT на 3-летнем горизонте с 3 сценариями.

## Уровень 3: предметная задача

**Когда:** содержательная задача в знакомом домене (госполитика,
молодёжная политика, форматы документов, конкретный проект).

**Что читать:** + `08_knowledge_maps.md` (предметные знания по
домену), `09_working_mechanisms.md` (рабочие механизмы для этого
класса задач).

**Пример:**
> Пользователь: «Напиши концепцию по нацпроекту Молодёжь и дети».
> Claude: уровень 3 → `km-002: нацпроект МиД, 8 ФП, куратор Кириенко` +
> `wm-012: концепция — обратная логика от целевых показателей` →
> Применяет обратную логику, опирается на знание ФП и куратора.

## Уровень 4: глубокий аудит

**Когда:** мета-задачи, аудит vault'а, ревизия, поиск противоречий.

**Что читать:** все остальные карты по релевантности:
- `03_projects_registry.md` — реестр проектов
- `05_global_glossary.md` — термины
- `06_lessons_learned.md` — уроки
- `07_global_entities.md` — люди / организации
- `10_error_corrections.md` — журнал ошибок
- `11_confidence_scoring.md` — калибровка уверенности
- `12_cross_project_graph.md` — граф связей (если собран)
- `13_self_reflection.md` — журнал самоаудитов
- `14_audit_log.md` — расхождения

**Пример:**
> Пользователь: «Проверь, нет ли противоречий между проектами».
> Claude: уровень 4 → `audit`-скилл + `14_audit_log.md` +
> `scripts/audit_contradictions.py` → полный аудит, отчёт.

## Эвристики выбора уровня

| Сигнал в запросе | Уровень |
|---|---|
| «привет / спасибо / как дела» | 0 |
| «короче / подробнее / суммаризуй» | 1 |
| «какой формат / каким стилем / как лучше» | 1-2 |
| «составь план / стратегию / концепцию» | 2-3 |
| «напиши по проекту X» | 3 |
| «по нацпроекту / нашему домену» | 3 |
| «что у нас есть / покажи состояние» | -1 |
| «проверь / аудит / ревизия» | 4 |
| «сколько активных / статистика» | -1 или 0 |
| «покажи граф» | -1 (+ открыть `graph.html`) |

## Прерывание и апгрейд

Если Claude начал на уровне 1 и понял, что контекста мало — может
**подняться на уровень выше** без повторного чтения. Принцип:
прочитанное сохраняется в контексте, добавляется только новый слой.

Обратное (спуск на уровень ниже после чтения выше) не делается —
бессмысленно.

## Интеграция с action-скиллами

| Action-скилл | Стартовый уровень |
|---|---|
| `status` | -1 |
| `init` | 0 (создаёт карты) |
| `migrate` | 0 → 4 (полный разбор структуры) |
| `graph` | -1 |
| `backlinks` | 0 |
| `reflect` | 2-3 |
| `calibrate` | 4 |
| `archive` | 4 |
| `patterns` | 2 |
| `audit` | 4 |
