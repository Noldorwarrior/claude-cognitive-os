# status — Сводка состояния воркспейса

## Назначение

Быстро показать пользователю, что происходит в когнитивном воркспейсе:
какие карточки активны, сколько записей, какие пороги близки к срабатыванию,
какие задачи открыты.

Команда **read-only**: ничего не изменяет, только читает.

## Когда запускается

- Пользователь явно просит «статус» / «что в воркспейсе» / «состояние».
- После [[init]] или [[migrate]] — автоматически.
- Начало новой сессии с плагином (опционально, по user-preferences).
- Перед запуском крупной задачи — быстрый обзор контекста.

## Когда НЕ запускается

- Нет воркспейса → предложить [[init]].
- Пользователь просит детальный отчёт → делегировать в [[reflect]]
  или [[audit]].

## Workflow

### 1. Обнаружение воркспейса

- Читает `~/.claude/cognitive-os-workspaces.json`.
- Если несколько — берёт `active: true` или спрашивает.

### 2. Чтение 00_index.md

Источник всех счётчиков и статусов карточек.

### 3. Сбор реального состояния (sanity check)

- Сравнивает счётчики в `00_index.md` с фактическими записями в каждой
  карточке.
- Расхождение > 0 → предупреждение + предложение запустить
  `sync_check.py`.

### 5. Запись в лог (опционально)

Сам по себе status не пишет, но если обнаружены расхождения — предлагает
зафиксировать их в `14_audit_log.md` через отдельный шаг.

## Режимы

### `status` (default)

Полная сводка как выше.

### `status --short`

Только счётчики + открытые вопросы. Для быстрой проверки.

### `status --porogi`

Только пороги и близкие срабатывания.

### `status --active`

Только активные карточки, без dormant.

### `status --domain=<domain>`

Фильтр по домену (`govsector-documents`, `presentations` и т.д.).

## Интеграции

- [[cog-detector]] (subagent) — если status обнаружил, что порог близок
  к срабатыванию (2 из 3) — cog-detector подготавливается к активации.
- [[productivity:task-management]] — если есть `TASKS.md`, открытые
  задачи тоже показываются в сводке.
- [[reflect]] — предложение запустить, если давно не было.
- [[calibrate]] — предложение запустить по расписанию.
- [[audit]] — предложение запустить, если открытых audit-XXX много.

## Верификация

**Не требуется** — read-only команда. Но сама может триггерить запуск
верификации, если обнаружены рассинхроны счётчиков (sync_check.py).

## Частые ошибки

1. **Устаревший `00_index.md`** — счётчики не совпадают с реальными.
   Защита: шаг 3 (sanity check) + предложение sync_check.
2. **Dormant карточки игнорируются** — пользователь может не знать, что
   они существуют. Защита: режим `--active` по умолчанию, полный вывод
   показывает dormant отдельно.
3. **Фильтр по домену** не находит записей. Защита: показать список
   существующих доменов из `11_confidence_scoring.md`.
