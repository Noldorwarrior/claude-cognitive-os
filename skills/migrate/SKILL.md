# migrate — Миграция воркспейса

## Назначение

Перенести содержимое когнитивного воркспейса из старой версии в новую без
потери данных. Основной сценарий: миграция из skill `cognitive-os v1.0.1`
(18 карт master_index) в plugin `claude-cognitive-os v1.3` (15 карточек
00-14).

## Когда запускается

- У пользователя есть старый воркспейс v1.0.1 (обнаруживается по
  `master_index.md` или 18 карточкам с именами без нумерации 00-14).
- Выходит новая версия плагина с изменёнными карточками или порогами.
- Пользователь явно говорит «мигрируй» / «обнови структуру».

## Когда НЕ запускается

- Воркспейса нет → [[init]].
- Воркспейс уже v1.3 и актуален → [[status]].
- Пользователь хочет слить два воркспейса в один → это отдельный кейс,
  делегировать [[cog-archivist]] + ручная сверка.

## Workflow

### 1. Обнаружение источника

Автоопределение версии:

| Сигнал | Версия |
|---|---|
| `master_index.md` + 18 карт без нумерации | skill v1.0.1 |
| `00_index.md` + 15 карт 00-14 | plugin v1.3 |
| `00_index.md` + другая структура | custom (остановиться, спросить) |

### 2. Создание бэкапа

**ОБЯЗАТЕЛЬНО** перед любыми изменениями:
```
cognitive-os-backup-{{DATE}}/
```
Полная копия старого воркспейса. Без этого — отказ миграции.

### 3. Маппинг v1.0.1 → v1.3

| v1.0.1 (18 карт) | v1.3 (15 карточек) | Комментарий |
|---|---|---|
| `master_index.md` | `00_index.md` | Перенос, переструктурирование |
| `user_profile.md` | `01_user_profile.md` | Прямой перенос |
| `patterns.md` | `02_patterns.md` | Прямой + добавить frontmatter thresholds |
| `projects.md` | `03_projects_registry.md` | + фолдер `projects/` |
| `meta_decisions.md` | `04_meta_decisions.md` | Прямой |
| `glossary.md` | `05_global_glossary.md` | Прямой |
| `lessons.md` | `06_lessons_learned.md` | Прямой |
| `entities.md` | `07_global_entities.md` | Прямой |
| `knowledge_maps.md` | `08_knowledge_maps.md` | Прямой |
| `mechanisms.md` | `09_working_mechanisms.md` | Прямой |
| `errors.md` | `10_error_corrections.md` | Прямой + mandatory П5 |
| `confidence.md` | `11_confidence_scoring.md` | Прямой |
| `cross_project.md` | `12_cross_project_graph.md` | Прямой |
| `reflection.md` | `13_self_reflection.md` | Прямой |
| `audit.md` | `14_audit_log.md` | Прямой |
| `calibration.md` | → в `11_confidence_scoring.md` § «Калибровочные рекомендации» | Merge |
| `templates.md` | → вынесено в `${CLAUDE_PLUGIN_ROOT}/templates/` | Удалить из воркспейса |
| `integration.md` | → в `skills/cognitive-os-core/references/integration_with_verification.md` | Удалить из воркспейса |

### 4. Трансформации контента

**Wikilinks:** проверить и обновить ссылки:
- `[[patterns]]` → `[[02_patterns]]`
- `[[errors]]` → `[[10_error_corrections]]`
- И т.д. по маппингу.

**Frontmatter:** добавить новые поля:
- `thresholds:` там, где их не было.
- `version: "1.0"` в каждую карточку.
- Счётчики (`active_count`, `archived_count`).

**IDs:** сохранить существующие (pat-001, ec-005 и т.д.), нумерацию не
перестраивать.

### 5. Валидация миграции

- Каждая запись из v1.0.1 должна иметь соответствие в v1.3.
- Никакие ID не потеряны.
- Wikilinks не сломаны.
- Frontmatter валидный YAML.

### 6. Diff-отчёт

Сформировать `docs/migration_report_{{DATE}}.md`:
- Что перенесено: количество записей по каждой карточке.
- Что изменено: новые поля, новые правила.
- Что отдельного внимания: карточки, переведённые в `dormant`
  (10-14 активируются по порогам).
- Что потеряно: **должно быть 0**. Если не 0 — откат миграции.

### 7. Rebuild `_generated/`

Перегенерировать:
- `_generated/graph.html` ([[graph]]).
- `_generated/backlinks.md` ([[backlinks]]).
- `_generated/graph.mermaid` ([[graph]]).

### 8. Верификация П14 «Итератор»

Обязательно для миграции (критичный класс задач):
- defense_against_regression (М25) — ничего не потерялось.
- meaning_drift (М26) — определения не сместились.
- diff_before_after (М24) — сверка структур.
- input_output_match (М21) — все записи сохранены.
- file_consistency (М22) — карточки согласованы.

Результат → `docs/verification_log.md`.

## Rollback-план

Если миграция провалила валидацию:
1. Остановить запись в новый воркспейс.
2. Сохранить `docs/migration_error.md` с подробностями.
3. Пользователь работает из `cognitive-os-backup-{{DATE}}/`.
4. Предложить ручную миграцию с указанием конкретных проблем.

## Команды

| Команда | Действие |
|---|---|
| `мигрируй воркспейс` | Запустить migrate |
| `мигрируй v1.0.1 → v1.3` | Принудительно |
| `migrate cognitive os` | Алиас |
| `покажи план миграции` | Dry-run без записи |
| `откати миграцию` | Восстановление из бэкапа |

## Интеграции

- [[cog-archivist]] (subagent) — помогает с merge `calibration.md` в
  `11_confidence_scoring.md`.
- [[verification]] плагин — пресет П14.
- [[status]] — запускается после миграции для показа результата.
- [[graph]], [[backlinks]] — rebuild `_generated/`.

## Частые ошибки

1. **Пропущен бэкап** → критическая ошибка. Защита: шаг 2 обязателен.
2. **Сломанные wikilinks** после переименования. Защита: М22 в П14.
3. **Потеря IDs** при переструктурировании. Защита: М21 + diff-отчёт.
4. **Двойная миграция** (запустили дважды). Защита: детекция версии в шаге 1.
