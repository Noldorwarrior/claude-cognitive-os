# Obsidian setup

> **Статус:** 🚧 Заглушка. Будет развёрнута в v1.4.0.

## Назначение

Пошаговая инструкция настройки Obsidian для комбо-сценария (плагин +
Obsidian работают с одной файловой системой). Охватывает горячие
клавиши, граф-вью, исключения, плагины community, темы.

## Планируемое содержание

- **Установка Obsidian** — ссылки на официальные сборки (desktop, mobile).
- **Настройка vault'а** — открытие `~/Documents/Claude/` как vault.
- **Критичные настройки** (см. README, раздел «Настройки Obsidian»):
  - Место для новых заметок = «Та же папка, что и текущий файл».
  - Всегда обновлять внутренние ссылки = ВКЛ.
  - Исключённые файлы: `Claude/Plugins`, `**/_generated`, `**/.claude-plugin`.
  - Wiki-ссылки `[[...]]` = ВКЛ.
- **Рекомендуемые community-плагины**:
  - Dataview — запросы по YAML frontmatter.
  - Templater — шаблоны карточек при создании.
  - Tag Wrangler — управление тегами.
  - Advanced Tables — редактирование markdown-таблиц.
- **Темы**: рекомендованные тёмные/светлые темы для чтения карточек.
- **Graph View настройка**: фильтры по типу сущности, цветовая схема
  совпадающая с `_generated/graph.html`.
- **Синхронизация между устройствами**: Obsidian Sync vs iCloud vs
  Syncthing — сравнение для mobile-сценария.
- **Конфликты с плагином**: какие файлы НЕ трогать в Obsidian
  (`_generated/`, `.claude-plugin/`, `projects/<name>/_generated/`).

## Текущее поведение (v1.3.3)

- `examples/starter_workspace/.obsidian/` содержит базовый конфиг с
  включённым Graph View и настройкой исключений.
- `init` скилл опционально создаёт `.obsidian/` при выборе
  пользователя (шаг диалога «Обсидиан-интеграция: да/позже»).
