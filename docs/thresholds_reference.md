# Справочник порогов v1.3

Все пороги хранятся в [[references/thresholds|thresholds.md]] внутри
скилла `cognitive-os-core`. Этот документ — развёрнутый комментарий
к ним.

## Детектор сигналов

### `pattern`

- **Значение по умолчанию:** 3
- **Что означает:** минимальное число похожих наблюдений (observations)
  для промоушена в `pat-NNN`.
- **Где применяется:** `cog-detector`, скилл `[[patterns]]`.
- **Когда повышать:** ложных срабатываний слишком много (> 30% от
  подтверждённых).
- **Когда понижать:** редкий домен, мало сигналов.

### `pattern_window_days`

- **Значение по умолчанию:** 30
- **Что означает:** окно дней, в котором считаем «похожие» наблюдения.
- **Где применяется:** `run_detector.py`.
- **Когда повышать:** сигналы в домене редкие, нужно растянуть окно.
- **Когда понижать:** рабочий цикл ускорился.

### `cluster_min_projects`

- **Значение по умолчанию:** 3
- **Что означает:** минимальное число проектов одного домена для
  выделения кластера.
- **Где применяется:** `cog-detector`, `12_cross_project_graph.md`.
- **Когда повышать:** много мелких кластеров, хочется только крупные.
- **Когда понижать:** маленький воркспейс.

### `systemic_error`

- **Значение по умолчанию:** 3
- **Что означает:** минимальное число ошибок одного типа в
  `10_error_corrections.md` для сигнала `systemic_error`.
- **Где применяется:** `cog-detector`.
- **Когда повышать:** новая команда, ошибки накапливаются быстро.
- **Когда понижать:** зрелая команда, ошибки редки.

### `similarity_threshold`

- **Значение по умолчанию:** 0.6 (Jaccard)
- **Что означает:** порог пересечения ключевых терминов для
  `conflict_suspect`.
- **Где применяется:** `audit_contradictions.py`, `cog-detector`.
- **Когда повышать:** слишком много ложных конфликтов.
- **Когда понижать:** подозреваем, что настоящие конфликты пропускаются.

### `threshold_near_trigger_percent`

- **Значение по умолчанию:** 0.80
- **Что означает:** уровень (от порога), при достижении которого
  генерируется сигнал «рядом с триггером».
- **Где применяется:** `cog-detector`.

### `cold_candidate_idle_months`

- **Значение по умолчанию:** 4.8 (80% от 6)
- **Что означает:** простой записи (no update_stats updates) для
  попадания в «холодные кандидаты».
- **Где применяется:** `cog-detector`, `[[archive]]`.

## Уверенность и калибровка

### `confidence_min_tasks`

- **Значение по умолчанию:** 5
- **Что означает:** минимум задач в домене для выдачи рекомендации
  калибровки.
- **Где применяется:** `calibrate_thresholds.py`.

### `overconfidence_threshold`

- **Значение по умолчанию:** 0.60
- **Что означает:** если > 60% задач домена имеют `predicted > actual`,
  сигнал overconfidence.
- **Где применяется:** `calibrate_thresholds.py`.

### `underconfidence_threshold`

- **Значение по умолчанию:** 0.60
- **Что означает:** то же, но в другую сторону.

### `critical_actual_threshold`

- **Значение по умолчанию:** 0.70
- **Что означает:** если средний actual в домене < 0.70 — critical
  underperformance, поднять пресет до П5.
- **Где применяется:** `calibrate_thresholds.py`.

### `well_calibrated_mae`

- **Значение по умолчанию:** 0.10
- **Что означает:** если MAE < 0.10, домен считается
  «хорошо откалиброванным», рекомендации не выдаются.

### `observation_to_pattern`

### `pattern_to_wm`

- **Минимум применений:** 3

### `wm_to_md`

- **Минимум применений:** 10

## Как менять пороги

### 1. Через скилл `[[calibrate]]`

Скрипт предлагает изменения, пользователь одобряет. Результат попадает в
`references/thresholds.md` как новый дефолт + версия фиксируется в
`04_meta_decisions` (md-calibration-NNN).

### 2. Вручную

Отредактируйте `references/thresholds.md`. Обязательно:
- Создайте `md-NNN` в `04_meta_decisions` с обоснованием.
- Добавьте `audit-NNN` типа `threshold_change`.
- Запустите `sync_check.py` для верификации.

### 3. Per-domain override

Формат в `references/thresholds.md`:

```markdown
## domain_override.design

- pattern: 5  # вместо 3
- cluster_min_projects: 5
```
