# Справочник пресетов верификации v1.3

Плагин `claude-cognitive-os` использует пресеты верификации для
проверки результатов работы. Пресеты делегируются во внешний плагин
[[verification]] (приоритетный путь) или выполняются во внутреннем
fallback через субагента `cog-verifier`.

## Малая верификация (М1-М5)

Быстрые проверки, выполняются автоматически в большинстве скиллов.

### М1 — Базовая

- **Механизмы:** №1 (точный перенос цифр/дат/имён), №2 (проверка
  выполнения всех пунктов запроса).
- **Когда применять:** любой короткий результат, где есть факты.
- **Время:** < 30 сек.
- **Кто использует:** `[[init]]`, `[[status]]`, `[[graph]]`.

### М2 — Расширенная

- **Механизмы:** М1 + №3 (сверка сумм), №4 (проверка границ), №5
  (формат документа).
- **Когда применять:** результаты с расчётами, числами, бюджетами.
- **Время:** 1-2 минуты.
- **Кто использует:** `[[calibrate]]`.

### М3 — Максимальная малая

- **Механизмы:** М2 + №6 (хронология), №7 (поиск противоречий).
- **Когда применять:** отчёты, хронологические документы.
- **Время:** 2-3 минуты.
- **Кто использует:** `[[audit]]`.

### М4 — Презентационная

- **Механизмы:** М1 + №8 (формат слайдов), №9 (согласованность
  pptx/html), №4 (проверка границ).
- **Когда применять:** презентации.
- **Время:** 2 минуты.
- **Кто использует:** (из других плагинов — events-presentations).

### М5 — Стратегическая малая

- **Механизмы:** М1 + №10 (скрытые допущения), №11 (парадоксы).
- **Когда применять:** стратегические выводы, короткие инсайты.
- **Время:** 2 минуты.
- **Кто использует:** `[[reflect]]` (частично).

## Полная верификация (П1-П14)

Архитектурные пресеты с большим набором механизмов.

### П1 — Аналитик

- **Механизмы (9):** №12 (обратная логика), №13 (декомпозиция фактов),
  №14 (оценка уверенности), №7 (противоречия), №15 (полнота), №16
  (спор «за/против»), №10 (скрытые допущения), №17 (граф

- **Когда применять:** аналитические записки, справки, обоснования.

## Механизмы (§1-§32) — справочник

### Фактологические (№1-7)

| № | Имя | Что проверяет |
|---|---|---|
| 1 | Точный перенос | Цифры, даты, имена совпадают с источником |
| 2 | Выполнение запроса | Все пункты запроса закрыты |
| 3 | Сверка сумм | Total = sum(parts) |
| 4 | Проверка границ | Значения в допустимых пределах |
| 5 | Формат документа | Структура, поля, стили |
| 6 | Хронология | События в правильном порядке |
| 7 | Противоречия | Нет взаимоисключающих утверждений |

### Документные (№8-9, №21-26, №29, №32)

| № | Имя | Что проверяет |
|---|---|---|
| 8 | Формат слайдов | Слайды соответствуют шаблону |
| 9 | Согласованность pptx/html | Два формата идентичны |
| 21 | Сверка вход-выход | Выход содержит всё из входа |
| 22 | Согласованность файлов | Counters, ссылки между файлами |
| 23 | Метаморфическое тест. | Инварианты сохраняются при вариациях |
| 24 | Diff было/стало | Изменения соответствуют плану |
| 25 | Защита от регрессии | Старые свойства не ломаются |
| 26 | Дрейф смысла | Определения сохраняются между версиями |
| 29 | Кросс-модальная | Текст+картинка/таблица+диаграмма сходятся |
| 32 | Ссылочная целостность | Все ссылки резолвятся |

### Логические (№10-17, №30)

| № | Имя | Что проверяет |
|---|---|---|
| 10 | Скрытые допущения | Явно выявлены недосказанные предпосылки |

## Делегирование — как работает

### Путь А (приоритетный): внешний плагин `verification`

Если плагин установлен:

```
cog-verifier → verification:verify --preset П14 --ids ... --context "..."
```

Внешний плагин выполняет всю методологию и возвращает структурированный
отчёт. `cog-verifier` только оборачивает результат в формат плагина.

### Путь Б (fallback): внутренняя реализация

Если плагин недоступен, `cog-verifier` выполняет упрощённый набор
механизмов своими силами:

- **П5:** 10 ключевых из 32 (не все 32).
- **П12:** полный список (9 механизмов).
- **П13:** полный список (8 механизмов).
- **П14:** полный список (8 механизмов).
- **П7:** 8 механизмов.
- **М1:** 2 механизма.

Внутренний fallback использует только информацию из воркспейса — не
выходит за его пределы.

### Флаг рекурсии

При вызове верификации устанавливается `in_verification=true`. Это
блокирует дальнейшие вызовы `cog-verifier` внутри уже запущенной
верификации, предотвращая бесконечные циклы.
