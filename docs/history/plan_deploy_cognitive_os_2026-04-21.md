## Варианты развёртывания

### A — только пустые оболочки (init)
- В каждой папке запускается скилл `init`
- Создаётся структура из шаблонов
- Карточки 02, 06, 10, 13, 14 остаются пустыми
- Карточки 01, 03, 04, 05, 07, 09 — заполняются постепенно в работе
- **Объём работы:** ~20-40k токенов, 12 проектов × init, 1 сессия
- **Результат:** 12 пустых vault, ждущих наполнения

### B — оболочка + перенос CLAUDE.md (рекомендуемый)
- В каждой папке: `init` → чтение CLAUDE.md → разнесение известных фактов по карточкам:
  - Цель проекта → `03_project_registry` и `04_meta_decisions`
  - Статус → `03_project_registry`
  - Ключевые цифры → `03_project_registry`
  - Ключевые люди → `07_entities`
  - Источники → `03_project_registry` + backlinks
  - Механизмы → `09_mechanisms`
  - Специфика, острые углы → `09_mechanisms` + `10_error_corrections` (preemptive)
  - Артефакты → `03_project_registry`
- В конце по каждому — `status` для сводки
- **Объём работы:** ~80-150k+ токенов, разбиение на 4 этапа по 3 проекта
- **Результат:** 12 живых vault с реальным контекстом

### D — пилот на 1 проекте
- Разворачиваем только в одной папке (предлагаю `Мифи` или `ТрендСтудио-Холдинг` как крупнейшие)
- Проверяем результат
- Тиражируем выбранный вариант на остальные 11

## Верификация — П13 «Аудитор» (полная)

По завершении всех этапов запускается пресет П13 с механизмами:

- №19 **цепочка происхождения** — каждый факт в карточке имеет источник (CLAUDE.md, строка/блок)
- №13 **декомпозиция фактов** — крупные абзацы из CLAUDE.md разложены на атомарные утверждения
- №18 **триангуляция** — факты из CLAUDE.md сверены с реальным состоянием папки (размер, наличие подпапок)
- №28 **эпистемический статус** — где знание зафиксировано как факт, где как гипотеза, где как план
- №11 **парадоксы** — поиск противоречий между карточками внутри одного vault
- №10 **скрытые допущения** — что мы приняли, но не проверили (например, «проект активен» без подтверждения)
- №23 **метаморфическое тестирование** — одна и та же сущность в разных vault не должна противоречить себе
- №4 **проверка границ** — пустые карточки помечены явно как пустые, не скрытые ошибки разноса

Дополнительно — **малая М3**:
- М1 точный перенос (цифры, даты, имена из CLAUDE.md)
- М2 проверка границ и согласованности формата карточек
- М3 хронология и поиск противоречий

**Результаты верификации** — отдельный блок в финальном ответе: что проверено, что найдено, уровень уверенности по каждому проекту.

## Острые углы и риски

1. **Переменный размер CLAUDE.md** — у некоторых проектов CLAUDE.md может быть подробным, у других отсутствовать. Если нет CLAUDE.md — создаём пустой vault (вариант A) для конкретной папки и помечаем как «требует наполнения».
2. **Дубли имён сущностей** — один человек может фигурировать в нескольких проектах. На этапе 5 делаем кросс-проектную сверку.
3. **Секретные/чувствительные данные** — CLAUDE.md может содержать ФИО, должности. Это нормально для внутреннего vault, но важно не тянуть их в общие карточки без необходимости.
4. **Размер проекта `Супермемори`** — исключён. Возвращаемся к нему отдельно после определения назначения.
5. **«Базовые правила работы»** — возможно это не проект, а системная папка. Уточнить перед развёртыванием.
6. **Объём за один ответ** — 12 проектов разом не уложатся. Поэтому этапами по 3 проекта.
