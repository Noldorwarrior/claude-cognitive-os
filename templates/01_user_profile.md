# 01 — Профиль пользователя

Карта стиля работы, предпочтений, ключевых правил. Формируется из
10 правил пользователя (Cowork user_preferences) и дополняется
наблюдениями Claude по ходу взаимодействия.

## Базовые данные

| Поле | Значение |
|---|---|
| Имя | rakhman |
| Язык общения | Русский (правило #2) |

| Роль | Руководитель / аналитик |

## Стиль работы

- **Формат docx по умолчанию:** A4 книжная, поля 2/2/3/1.5, Times New Roman
  14pt, красная строка 1.5см, межстрочный 1.15. Заголовки #0070C0.
  Подробнее — правило #6.
- **Язык общения:** русский, всегда (правило #2).
- **Методология перед работой:** при аналитических задачах — сначала
  кратко описать методологию, предложить альтернативы, получить
  одобрение (правило #8, #9).
- **Верификация:** по правилу #10 при комплексных задачах спрашивать
  через ask_user_input уровень верификации (Нет / Малая / Полная).

## Предпочтения по форматам

| Формат | Предпочтения |
|---|---|
| docx | Правило #6 (стандарт оформления) |
| pptx + HTML | Правило #7 (сначала pptx с уточнениями, потом интерактивный HTML 16:9) |
| Аналитическая записка / справка | Правило #9 (посыл → методология → уточнения → работа) |
| Промт | Правило #1 (триггер «ПРОМТ — », workflow с уточнениями) |

## Источники информации

- **Госполитика РФ:** приоритет официальным российским источникам —
  ТАСС, РИА, Кремлин.ру (правило #5).
- **Технические вопросы:** документация, web search.

## Паттерны запросов (связь с [[02_patterns]])

- Триггер `ПРОМТ — ` → workflow создания промта с итерациями.
- Запрос на аналитику → методология через [[govdoc-analytics:memo]].
- Запрос на презентацию → двухэтапный pptx + HTML.

## Ключевые правила (референс)

1. Триггер «ПРОМТ — ».
2. Всегда на русском.
3. Предварительная оценка объёма.
4. При создании документов — оценка объёма, разбивка на части.
5. Госполитика РФ → официальные источники.
6. Стандарт docx.
7. Двухэтапная презентация (pptx + HTML).
8. Методология перед работой.
9. Аналитические документы: посыл → методология → уточнения.
10. Верификация при комплексных задачах (Нет / Малая / Полная, М1–М5, П1–П14).
