# 10 — Журнал исправлений ошибок

Фиксация ошибок Claude и механизмов защиты от их повторения.

**Что попадает сюда:**
- Claude допустил ошибку и исправил (сам или по указанию).
- Пользователь явно указал на ошибку / расхождение.
- Верификация П5/П13/П14 выявила проблему.

**Что НЕ попадает:**
- Разовые опечатки, исправленные сразу (без последствий).
- Стилистические правки от пользователя (→ [[02_patterns]]).

## Категории ошибок и защитных механизмов

| Категория | Что значит | Защитный механизм |
|---|---|---|
| `factual` | Неверный факт / цифра / дата | Верификация М1 (точный перенос) |
| `methodological` | Неверный подход / методология | Обязательное согласование перед работой (правило #9) |
| `format` | Нарушен формат (docx, pptx) | Проверка М5 / М8 |
| `linguistic` | Неправильный язык / стиль | Сверка с [[01_user_profile]] |
| `scope` | Вышел за рамки запроса / упустил пункты | М2 проверка выполнения запроса |
| `arithmetic` | Ошибка расчёта | М3 сверка сумм, М20 двойной расчёт |
| `source_integrity` | Неверный источник / отсутствие ссылки | М18 триангуляция, М19 цепочка происхождения |
| `audience` | Несоответствие аудитории | М27 моделирование аудитории |
| `hallucination` | Выдумал факт / ссылку / данные | Триангуляция + требование источников |
| `drift` | Дрейф смысла при итерациях | М26 дрейф смысла, М25 защита от регрессии |
| `contradiction` | Противоречие внутри документа | М7 поиск противоречий |

## Системные ошибки (systemic)

Ошибка получает тип `systemic` при:
- **`thresholds.systemic_error`+ повторений** (default 3) в разных контекстах
- ИЛИ категория `critical` с одним случаем (высокий риск повторения)

Systemic-ошибка → **обязательный механизм защиты** в
[[09_working_mechanisms]] + запись в [[04_meta_decisions]] как принцип.

## Жизненный цикл записи

```
open (обнаружена)
  ↓
analyzed (категория + severity + причина определены)
  ↓
protected (механизм защиты создан в wm-XXX)
  ↓
closed_with_protection (верификация П5 прошла)
  ↓
[recurring — если ошибка повторилась несмотря на механизм]
  ↓
revised (механизм пересмотрен или создан дополнительный)
```

## Правила записи

1. **ID уникальный** `ec-NNN`.
2. **Категория из enum.**
3. **Severity обязателен.**
4. **Первопричина явно указана** — не «опечатка», а корневая причина.
5. **Механизм защиты** создан (или явно указано, что не нужен).
6. **Верификация П5** перед закрытием.
7. **Trigger pattern** — как Claude узнает о риске в будущем.

## Связь с верификацией (внешний плагин)

Каждая запись `ec-NNN` → обязательная верификация **пресет П5 «Максимум»**
(все 32 механизма). Это единственная карта с обязательным П5.

Результат верификации → в `docs/verification_log.md` плагина.

## Rate limit

Без rate limit — фиксируются ВСЕ ошибки, даже если их много. Архивация
по критерию: `closed_with_protection` + `last_seen` > 12 месяцев.

## Recurring-режим

Если ошибка со статусом `closed_with_protection` повторяется:
1. Счётчик `recurrence` +1.
2. Статус → `recurring`.
3. Trigger: запуск [[cog-verifier]] с пресетом П14 (итератор) для
   пересмотра механизма защиты.
4. Уведомление пользователю — **обязательно** (даже если бы не было
   по правилам).

## Команды

| Команда | Действие |
|---|---|
| `покажи ошибки` | Все открытые |
| `покажи systemic ошибки` | Только systemic |
| `покажи ошибки категории X` | Фильтр |
| `зафиксируй ошибку: <описание>` | Новая запись |
| `закрой ec-X` | Закрытие (с проверкой П5) |
