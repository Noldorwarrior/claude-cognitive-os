# 03 — Реестр проектов

Единый реестр всех проектов пользователя. Источник правды для
cross-project операций (поиск связей, аудит, граф).

**Принцип:** запись создаётся, когда в проекте накопилось 3+ задач
(триггер от [[productivity:memory-management]]) или когда пользователь
явно запросил `зарегистрируй проект`.

## Критерии записи

1. **Минимум 3 задачи** в [[productivity:memory-management]] TASKS.md
   ИЛИ явная команда `зарегистрируй проект`.
2. **Уникальный код** проекта (`proj-NNN`).
3. **Папка существует** в `Projects/<name>/`.
4. **Связь с сущностями** — минимум 1 ссылка на [[07_global_entities]].

## Интеграции

- [[productivity:memory-management]] — триггер регистрации при 3+ задач.
- [[project-knowledge-maps]] — каждый проект может иметь локальные карты
  (actors, goals, decisions, risks). Локальные знания НЕ дублируются
  здесь.
- [[08_knowledge_maps]] — если знание из проекта становится сквозным
  (3+ обращений из разных проектов) — переносится в карты знаний.

## Команды

| Команда | Действие |
|---|---|
| `покажи проекты` | Показать активные |
| `зарегистрируй проект: <данные>` | Создать новую запись |
| `переведи proj-X в completed` | Смена статуса |
| `архивируй proj-X` | Перенос в `_archive/` |
| `покажи граф проектов` | Открыть [[12_cross_project_graph]] |

## Связанные карты

- [[07_global_entities]] — люди и организации, привязанные к проектам.
- [[08_knowledge_maps]] — предметные знания, используемые в проектах.
- [[12_cross_project_graph]] — граф связей между проектами.
- [[productivity:memory-management]] — задачи по проектам (слой 2).

---

**Принцип не-дублирования:** здесь хранятся только **метаданные**
проекта. Содержательные данные (актёры, цели, решения, риски) живут
в папке проекта как локальные карты.
