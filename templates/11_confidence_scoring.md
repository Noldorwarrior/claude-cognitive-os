# 11 — Калибровка уверенности

Журнал калибровки уверенности Claude по доменам / классам задач.
Связывает **заявленную уверенность** с **фактическим результатом** для
каждой задачи.

**Когда создаётся карта:** при накоплении
`thresholds.confidence_min_tasks`+ задач в одном домене (default 5).

**Зачем:**
- Понимать, где Claude чрезмерно уверен (overconfident).
- Понимать, где Claude недооценивает (underconfident).
- Автокалибровать пороги (`cognitive-os-calibration`).
- Адаптировать вывод (если уверенность реально низка — просить
  верификацию).

## Протокол калибровки (для каждой задачи)

После завершения задачи (через [[govdoc-analytics:self-reflection]]):

1. **Заявленная уверенность** — что Claude заявил перед началом
   (0–100%).
2. **Фактическая успешность** — что оценил сам + что подтвердил
   пользователь (0–100%).
3. **Расхождение** — (факт − заявл.).
4. **Ошибки** — ссылки на [[10_error_corrections]] если были.
5. **Инсайт** — что добавить в калибровку (если паттерн повторяется).

## Эпистемический статус записей

Любая запись может иметь маркер:
- `confirmed` — пользователь подтвердил успешность.
- `self-assessed` — Claude оценил сам.
- `inferred` — выведено из косвенных сигналов (не было явной
  реакции).

## Калибровочные рекомендации

Скрипт `calibrate_thresholds.py` сканирует карту и выдаёт рекомендации:

- Если **средний гэп > +10%** (overconfident) — снизить заявленную
  уверенность в домене.
- Если **средний гэп < −10%** (underconfident) — повысить.
- Если **расхождения высокие** (> 20% на задачу) — увеличить
  обязательную верификацию до П5.

Рекомендации → в [[04_meta_decisions]] как предложения, требующие
одобрения пользователя.

## Домены для калибровки (заготовки)

| Домен | Класс задач |
|---|---|
| `govsector-documents` | Аналитические записки, справки, концепции |
| `govsector-speech` | Речи, тезисы, выступления |
| `govsector-letters` | Официальные письма |
| `presentations` | Презентации (pptx + HTML) |
| `events` | Сценарии, реестры, чек-листы |
| `finance-analytics` | Финансовые модели, анализ |
| `data-analytics` | SQL, статистика, визуализация |
| `coding` | Скрипты, автоматизация |

## Правила записи

1. **Уникальный ID** `domain-NNN` для каждого домена.
2. **Минимум `thresholds.confidence_min_tasks` задач** в домене перед
   созданием.
3. **Журнал задач** — каждая запись с датой, заявленной и фактической
   уверенностью.
4. **Калибровка** — как минимум раз в месяц через action-скилл
   `calibrate`.

## Связь с верификацией

Если в домене средняя фактическая успешность < 70% → **обязательная
верификация П5** для новых задач в домене (пока не поднимется до ≥
80%).

## Команды

| Команда | Действие |
|---|---|
| `покажи калибровку` | Все домены |
| `покажи калибровку X` | Один домен |
| `откалибруй` | Запустить скрипт |
| `добавь задачу в калибровку: <данные>` | Новая запись вручную |
