# CLAUDE.md — bootstrap файл когнитивного воркспейса

> Этот файл загружается автоматически в начале каждой сессии Claude
> в этом воркспейсе. Он задаёт навигацию и контракт работы с
> когнитивной памятью.

## Что это за воркспейс

Когнитивный воркспейс плагина `claude-cognitive-os`: структурированная
память проекта в 15 карточках, автоматический граф связей, извлечение
паттернов из повторяющихся событий.

## Reading ladder

- **Level 0 (always)**: `00_index`, этот файл.
- **Level 1 (on demand)**: `01_user_profile`, `03_projects_registry`,
  `05_global_glossary`, `07_global_entities`.
- **Level 2 (task-specific)**: `02_patterns`, `06_lessons_learned`,
  `08_knowledge_maps`, `09_working_mechanisms`,
  `10_error_corrections`, `11_confidence_scoring`,
  `12_cross_project_graph`, `13_self_reflection`, `14_audit_log`.

## Контракт работы

1. **До ответа**: прочитай `00_index` и этот файл; при необходимости —
   карточки уровня 1/2 по теме запроса.
2. **Во время работы**: используй `[[wikilinks]]` при ссылке на
   карточки; сохраняй YAML frontmatter; пиши на русском.
3. **После задачи** (если задача комплексная: 2+ файлов/источников,
   аналитика, данные, документы, расчёты):
   - Запусти `reflect` для самоанализа.
   - Обнови `13_self_reflection` с оценкой 0–100% и уроками.
4. **При обнаружении повторяющегося паттерна (≥3 случаев)**:
   детектор сам предложит вынести его в `02_patterns` через
   `pattern_candidate`.
5. **При расхождении**: не перебивай первичный источник (например,
   версия в `.claude-plugin/plugin.json`) вторичным (README/CHANGELOG).

## Критичные пороги (см. `00_index` для актуальных)

- `pattern` = 3 (минимум случаев для паттерна)
- `pattern_window_days` = 30 (окно наблюдения)
- `systemic_threshold` = 3 (минимум для systemic error)
- `min_occurrences` = 3 (минимум для кластера)

Пороги автокалибруются через [[calibrate]] на основе реальных данных.
