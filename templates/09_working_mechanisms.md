# 09 — Рабочие механизмы

Компактные описания **способов решения класса задач**: что делать,
каким инструментом, в каком порядке, с каким результатом.

**Разница с паттерном:**
- [[02_patterns]] — наблюдение о поведении пользователя.
- Механизм — **рецепт работы Claude** для класса задач.

**Разница с мета-решением:**
- [[04_meta_decisions]] — принцип высокого уровня.
- Механизм — **конкретная процедура** реализации принципа.

## Правила записи

1. **Класс задач указан** — не частная задача, а класс.
2. **Источник из enum** (observation / error_correction / skill_adoption /
   user_rule / user_correction).
3. **Procedure конкретная** — шаги можно выполнить без интерпретации.
4. **Inputs / Outputs** формализованы.
5. **Уникальный ID** `wm-NNN`.
6. **Верификация** П12 или П14 перед активацией.

## Связь механизма с картой знаний

Механизм часто опирается на конкретную карту знаний:
- wm-XX (как писать аналитическую записку) → km-XX (структура АЗ).
- wm-YY (как обращаться к чиновнику) → km-YY (формы обращения в
  госсекторе).

При обновлении km-XX механизм wm-XX получает пометку `under_review` до
повторной проверки.

## Ревизия

- Каждый запуск механизма логируется в [[13_self_reflection]].
- При низкой успешности (<70% по [[11_confidence_scoring]]) — пометка
  `under_review`, при критической — `deprecated`.
- Устаревший механизм → supersession: `superseded_by wm-NNN`.

## Команды

| Команда | Действие |
|---|---|
| `покажи механизмы` | Активные |
| `покажи механизмы для X` | Фильтр по классу задач |
| `добавь механизм: <описание>` | Новая запись |
| `применить wm-X` | Выполнить procedure |
| `архивируй wm-X` | В `_archive/` |
