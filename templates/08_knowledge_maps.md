# 08 — Карты предметных знаний

Компактные «слепки» знаний по доменам / темам, используемым в 2+
проектах или с частым обращением. Каждая карта — самодостаточная
справка по теме.

**Когда создаётся:**
- Пользователь обратился к одному и тому же знанию
  `thresholds.knowledge_map_access`+ раз (default 3).
- Явный запрос `добавь знание про {домен}`.
- Импорт из внешнего источника с высоким приоритетом.

**Разница с [[05_global_glossary]]:**
- Глоссарий — термин + определение (короткое).
- Карта знаний — **развёрнутое знание** со структурой, связями,
  процедурами.

## Правила создания карты

1. **Порог обращений:** минимум `thresholds.knowledge_map_access` (default 3)
   ИЛИ явная команда.
2. **Уникальный ID** `km-NNN`.
3. **Домен обязателен.**
4. **Источники указаны** — ссылки на закон / указ / документ.
5. **Структура знания** — минимум 5 блоков (основные понятия, игроки,
   процесс, документы, подводные камни).
6. **Применение в работе** — конкретные связи с механизмами.
7. **Верификация** П13 «Аудитор» перед активацией.

## Принцип не-дублирования

- **Часть знания уже в [[govdoc-analytics:govdoc-context]]** → не
  копировать, только ссылка.
- **Часть знания в локальной карте проекта** → перенести в глобальную
  только при 2+ проекте-потребителе.
- **Термины** — всегда ссылка на [[05_global_glossary]], не переписывать.

## Ревизия и актуализация

- Каждые 6 месяцев (или по триггеру изменения закона / указа) — пометка
  `under_review` и вызов [[cog-verifier]] с пресетом П13.
- Новые источники → обновление, старые отправляются в `history` блок
  внутри карты.

## Принцип «живого» знания

Карта знаний — не статический документ, а **действующий инструмент**:
- Применяется в работе (что Claude должен сделать, прочитав).
- Обновляется при каждом новом источнике.
- Если 3+ попытки применения дали неверный результат → пометка
  `under_review`, переход в [[10_error_corrections]] с кодом
  ec-NNN-broken-knowledge.

## Команды

| Команда | Действие |
|---|---|
| `покажи знания` | Список всех карт |
| `покажи знание про X` | Открыть km-X |
| `добавь знание про X` | Начать создание новой |
| `обнови знание X` | Ревизия + источники |
| `архивируй знание X` | В `_archive/` |

**Триггеры записи:**
- `thresholds.knowledge_map_access`+ обращений к одной и той же теме.
- Явная команда `добавь знание про X`.
- Импорт из внешнего справочника.

**Триггеры чтения:**
- Reading ladder уровень 3 (предметная задача в знакомом домене).
- Запрос касается домена, по которому есть карта.
- Планирование работы по проекту → загрузить km по доменам проекта.
